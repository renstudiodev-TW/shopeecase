# Yahoo 拍賣大量上架轉檔工具

仁格數位科技工坊（Ren Studio）

## 用途

把 EC-Mart 大量下載助手匯出的「蝦皮商品 Excel」，轉成 Yahoo 大量刊登助手能直接吃的格式。
應施檢驗商品（電池、充電器、LED 燈等）會自動篩到「需人工處理」工作表，方便您後續補上 BSMI 字號。

## 系統需求

- Windows 10 / 11
- Python 3.10 或以上

## 安裝

第一次使用：
```
pip install -r requirements.txt
```

## 啟動

雙擊 `start.bat`，或在命令列執行：
```
python app.py
```

## 操作流程

1. **Step 1：選擇檔案** — 選 EC-Mart 匯出的 .xlsx
2. **Step 2：設定參數**
   - 各物流方式運費（請依您與物流商實際合約調整）
   - 應施檢驗判定關鍵字（可增刪）
   - 勾選「我了解 BSMI 判定責任」
3. **Step 3：預覽結果**
   - 看可上架、需處理、修正紀錄三個分頁
   - 確認沒問題後 → 匯出 Excel

## 輸出檔案結構

匯出的 .xlsx 包含三個工作表：
- **可上架商品** — 修正完成、可直接餵 Yahoo 大量刊登助手
- **需人工處理** — 應施檢驗商品 / 資料不全的商品（含「未通過原因」欄）
- **修正紀錄** — 每筆自動修正的詳細紀錄（原列號、欄位、原值、新值、規則）

## 修正規則總覽

| 規則 | 說明 |
|---|---|
| 收款 ↔ 運送對應 | 接受輕鬆付 X = yes 時，自動將對應運送方式設為運費金額 |
| 運費型別轉換 | 運送方式欄位從 yes/no 改為運費整數 |
| 套用全店運費 | 預設改為 no（避免依賴 Yahoo 後台設定） |
| 商品檢驗字號 | 若原檔無此欄位則自動新增 |
| 應施檢驗分流 | 命中關鍵字 → 移至「需人工處理」 |

## 法律提醒

BSMI 判定僅以**關鍵字輔助**。若您有應施檢驗商品被錯標為「商品無需檢驗」並上架販售，
依商品檢驗法可處 20–200 萬元罰鍰，責任由您（賣家）承擔。
建議：使用本工具後仍須逐筆檢視「可上架」工作表，確認沒有應施檢驗商品被誤分流。
