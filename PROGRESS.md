## 已完成

### 反向工程 Yahoo 大量刊登助手（5 大格式 GAP）

| GAP | 內容 | 修正規則 |
|---|---|---|
| 1 | 收款方式 vs 運送方式不一致 | 接受輕鬆付 X = yes → 對應運送方式自動補運費 |
| 2 | 運送方式錯填 yes/no | 改為運費整數（0–500） |
| 3 | 套用全店運費 = yes 依賴後台 | 預設改 no，商品自帶運費 |
| 4 | 「商品檢驗字號」不是獨立欄位 | 是「屬性欄位 1」的內容（依類別變動） |
| 5 | 屬性欄位 1 schema | `屬性名-:值`（單選） / `屬性名+:值1,值2`（複選） / `(none)`（無必填） |

### 工具實作（`yahoo_converter/`）

- **`converter.py`**：純函數核心邏輯，5 個 GAP + 類別樣板 + (none) 標記 + 保守分流
- **`app.py`**：CustomTkinter GUI，3 步驟流程 + 表格式類別樣板 + 4 層法律提醒
- **`start.bat`**：雙擊啟動腳本（純英文，符合 Windows bat 規則）
- **`README.md`** + **`requirements.txt`**

### 工具能力（截至昨晚）

- 1052 筆 / 32 類別檔案 → 修 4208+ 個金物流欄位
- 「全填測試字號」一鍵填占位符 + 自動為「無必填屬性」類別填 `(none)`
- 「儲存字號資料 / 填入已存在之字號」（JSON）— 第一次填好，未來新檔案一鍵套用
- 點擊「代表商品」按鈕看完整品名清單（彈窗，可複製商品名去查 BSMI/NCC）
- 智慧攔截：未填樣板 / 部分填寫 → 提醒對話框
- 結果頁綠色 banner（可上架=0 但有修正時）

## 已知未解決問題

### Yahoo 仍會回報 4 筆錯誤（用「全填測試字號」+ 不修正）

| 列 | 類別 | 工具填的 | Yahoo 真實要求 |
|---|---|---|---|
| 15 | 2092105294（LG 冷氣控制器）| `(none)` | `商品檢驗字號-:R12345` |
| 868 | 2092080007（e27 燈座）| `商品檢驗字號-:...` | `適用電壓-:110V` |
| 902 | 2092074989（華碩平板旅充）| `NCC型式認證碼-:...` | `商品檢驗字號-:R12345` |
| 903 | 同上 | 同上 | 同上 |

**根因**：商品名稱關鍵字推測在這 3 個類別失準。
- 2092105294「冷氣控制器」字面沒有 BSMI 關鍵字 → 工具誤判 (none)
- 2092080007「e27 燈座」需要的是「適用電壓」屬性，不是 BSMI
- 2092074989「平板旅充」被「平板」騙到推測 NCC（實際旅充才是商品本體 → BSMI）

**短期解法**：客戶在類別樣板表格中手動修正這 3 行。

**長期解法（明天要做）**：給工具「從 Yahoo 錯誤訊息學習」的能力。
